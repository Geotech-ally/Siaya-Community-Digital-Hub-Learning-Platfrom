# Contributing to Siaya Community Digital Hub Learning Platform

Thank you for contributing! This document explains how to get started, submit changes, and maintain code quality.

## Getting Started

1. Fork the repository.
2. Clone your fork locally.
3. Create a new branch for your work.

## Development Workflow

- Use feature branches for new work.
- Keep commits focused and descriptive.
- Open a pull request against the main branch.

## Code Style

- Backend code follows NestJS conventions.
- Frontend code follows Next.js and React best practices.
- Run linters before submitting changes:
  - `cd backend && npm run lint`
  - `cd frontend && npm run lint`

## Testing

- Run backend tests with `cd backend && npm test`.
- Add tests for bug fixes and new features.

## Documentation

- Update `README.md` or module-level docs when API or architecture changes.
- Add examples for new frontend components and backend routes.
